# 禁闭逃杀 · 电子版 v1.0（基本最终版 / 可公开试玩）

把《禁闭逃杀》纸面规则做成可在网页真实多人游玩的电子桌游：任何玩家打开邀请链接即可加入房间，9 人用手机或电脑参与，刷新可断线重连，核心规则自动结算，房主只需确认、纠错与推进。

> 规则唯一来源：`禁闭逃杀_规则手册.md`。所有规则数据与逻辑集中在 `src/game/`（config / engine / resolution / utils），不写死在 UI 组件里。地图使用项目内 `public/禁闭逃杀_地图.png`。

## 技术栈

- Next.js 14（App Router）+ TypeScript（strict）+ Tailwind CSS
- Zustand 状态管理；纯函数游戏引擎（与 UI、存储完全解耦）
- **可插拔存储适配层**：
  - **远程模式**（配置 Supabase 后）：Supabase Postgres 存整局 `GameRoom`（JSONB），Next.js 服务端 API + 乐观锁，客户端轮询同步，真实多人 + 断线重连 + Vercel 部署。
  - **本地模式**（未配置环境变量）：localStorage + 跨标签页 `storage` 事件，零配置即可单机/同浏览器热座测试。
- `tsx` 运行纯函数测试（`npm test`，共 44 项）

## 本地启动

```bash
npm install
npm run dev          # http://localhost:3000
```

不配置任何环境变量即以**本地模式**运行（适合一台电脑开多个标签页验收）。生产构建：`npm run build && npm start`。

## 环境变量

复制 `.env.example` 为 `.env.local` 并按需填写：

| 变量 | 作用 |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 项目 URL。**填了它就进入远程多人模式**，前端会改走服务端 API。 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 匿名公钥（当前服务端用 service role，此项预留，可填 anon key 或留空）。 |
| `SUPABASE_SERVICE_ROLE_KEY` | **仅服务端使用**，切勿暴露给前端。用于服务端读写数据库。 |

三者都不填 → 本地模式；填齐 `NEXT_PUBLIC_SUPABASE_URL` + `SUPABASE_SERVICE_ROLE_KEY` → 远程多人模式。

## 配置 Supabase（真实多人）

1. 登录 https://supabase.com → New project。
2. 进入 **SQL Editor**，粘贴执行本仓库的 `supabase/schema.sql`（建 `rooms` 表 + 触发器）。
3. 进入 **Project Settings → API**，复制 `Project URL`、`anon public`、`service_role`。
4. 在本地 `.env.local`（或 Vercel 环境变量）中填入对应三项。
5. 重启 `npm run dev`，首页底部提示会变为「真实多人模式」。

数据模型：整局 `GameRoom` 以一行 JSONB 存于 `rooms`，配 `rev` 乐观锁防止并发覆盖；秘密令牌 `tokens` 与阶段快照 `snapshots` 单列存放，不随公共数据下发。

## 部署到 Vercel

1. 把仓库推到 GitHub。
2. Vercel → New Project → 导入该仓库（框架自动识别 Next.js）。
3. 在 **Settings → Environment Variables** 填入上面三个变量（`SUPABASE_SERVICE_ROLE_KEY` 务必保密）。
4. Deploy。完成后用线上域名打开首页即可创建房间、生成邀请链接分享给朋友。
5. 之后在 Supabase 控制台执行过 `schema.sql` 即可正常使用。

> 未配置 Supabase 也能部署，但只会是本地模式（各浏览器数据互不相通），无法真正多人。

## 创建房间与邀请玩家

- **创建**：首页填房主昵称 →（可勾选开发调试模式允许少于 9 人）→ 创建。进入大厅得到 6 位房间码。
- **邀请**：大厅点「复制邀请链接」，把 `/room/房间码` 链接发给朋友；或直接口播房间码，朋友在首页「加入房间」输入。
- **加入**：打开链接 → 在空座位填昵称加入 → 设置职业 / 基因点（武力+速度+负重=10）/ 出生房间 → 准备。
- **身份与重连**：加入后浏览器保存身份令牌；刷新或换标签打开同一链接会自动回到你的座位。房主刷新后仍是房主。

## v1.0 功能总览

- **真实多人同步**：Supabase 持久化 + 轮询同步 + 乐观锁；房主/玩家令牌身份；刷新与断线重连；localStorage 本地兜底零配置可玩。
- **完整地图移动**（v0.3 基础上）：按速度可达计算、必须移动、激光室即时伤害、绳索/暗影上下楼、202 直升机、B105 垃圾管道、B403 传送室、肾上腺素提速；玩家页显示路径/步数/特殊移动/激光警告。
- **结算引擎**（严格按规则 8.1 固定顺序）：房间效果 → 战斗/乱斗/枪械压制 → 暗影吸血 → 火箭筒 → 毒气 → 水粮 → 道具回血/状态 → 死亡/复活。房主「生成预览 → 确认应用」。
- **职业技能**：14 职业配置化（`config/roles.ts` + `engine/roleEffects.ts`），自动化程度见下表。
- **交易系统**：自由阶段交易道具/顺位卡，发起 / 接受 / 拒绝 / 撤销，自动转移 + 负重提示 + 日志；房主可取消异常交易。
- **房主纠错**：生命 / 状态 / 位置 / 基因 / 道具 / 顺位卡 / 技能次数 / 毒气楼层 / 解毒房间 + 重置提交 + 公开日志。
- **快照 / 回滚 / 导出导入**：每次阶段切换自动快照（保留最近 5 个），房主可回滚；可导出 / 导入房间状态 JSON。
- **终局**：仅剩 1 人立即获胜 / 第 6 轮结束 / 全员暗影三种结束条件；金条兑换生命；规则 17.3/17.4 排名；金魔方积分 9..1；复盘见排名 + 公开日志 + 导出。

## 已知限制 / 后续建议

- **秘密信息为信任模型**：服务端按整份房间数据下发，未做服务端字段级隐藏（与纸面游戏的朋友间信任一致）。后续可加服务端按身份脱敏。
- 14 个职业均已自动化（部分「由玩家选择」的细节如慈善家夺取基因、预言家基因分配采用自动/房主辅助默认值，见下条）。
- 同步为轮询（约 1.5s）；如需更实时可接 Supabase Realtime。
- 慈善家夺取基因、预言家基因分配等「由玩家选择」的交互目前为自动/房主辅助，后续可做交互式选择弹窗。

# v1.0.1：规则漏洞修复与信息隐藏加固

> 本次不是重做项目，而是在 v1.0 基础上集中修复规则正确性、信息隐藏、行动顺序、地图步数、抽卡次数与关键职业技能漏洞。规则唯一来源仍为 `禁闭逃杀_规则手册.md`。

## 本次修复的漏洞

1. **角色选择去重 + 隐藏**：准备阶段只记录「想选的角色」(`preferredRoleId`)，他人不可见（只显示已提交/未提交）；开局统一解析，唯一者锁定、撞车者合并到同一池从剩余角色**无放回随机抽取**，保证人人唯一。
2. **地图步数精修**：规则层 `mapGraph.ts`（图结构算步数/连接/特殊移动）与展示层 `mapCoords.ts`（归一化坐标）彻底分离；新增 `/dev/map-checker` 校验页；不再用图片距离推断可达性。
3. **抽卡一次 + 即时私密展示**：每次行动只能常规抽卡一次，抽完立即向本人私密展示抽到内容；抽卡结果不进公共日志。
4. **「酒」全量改「果汁」**：道具内部 id 统一 `juice`，保留 `wine→juice` 别名与读档迁移；`B601 酒窖` 房名保留但产出果汁；骰子效果不变。
5. **饮品师多果汁多目标**：按使用瓶数生成多个目标条，每瓶独立目标 + 各自 3 骰面。
6. **玩家界面显示 ID + 角色名**：列表、技能目标、催眠/侦探目标均显示 `P# 昵称｜角色名`。
7. **严格顺位行动 + 行动锁**：只有当前顺位玩家可操作；提交移动后锁定移动/投票/技能，到达后可抽一次卡/用房间功能，点「结束本轮行动」后整轮锁死、轮到下一顺位。
8. **所有伤害延迟到结算**：激光室等不再在行动阶段即时扣血，统一在结算阶段「房间效果」步骤处理。
9. **催眠师隐藏 + 强制移动**：目标本人在轮到自己前不知情；按 5 步（无视速度、不可捷径）判定，成功后强制锁定终点、不能自选；不可达则催眠失败。
10. **行动面板状态隔离**：切换/轮换玩家时面板以 `玩家+轮次+阶段` 为 key 整体重挂载，清空上一名玩家的目标房间/毒气/抽卡/技能残留。
11. **私家侦探跟踪**：只能跟踪顺位在自己之前且已行动的玩家，不显示其去向；发动后直接前往其终点，无需再选房间。
12. **玩家列表按顺位排序**：play / board 均按本轮顺位卡升序显示。
13. **公开日志延迟到结算后**：行动阶段移动/抽卡/技能写为私密（仅本人可见），公共看板只显示结算后公开的信息。

## 完整鼠标验证清单

1. **角色撞车统一抽取**：本地热座建房，多个座位选同一角色并准备，开始游戏；看公共日志「角色分配完成」每人角色唯一。
2. **角色选择互相不可见**：准备阶段切到 A 身份能看到自己所选角色，他人座位只显示「角色：已选择（保密）」。

4. **必须按顺位**：非当前顺位玩家面板提示「还没轮到你行动」，提交被拒。
5. **结束后不可改**：当前玩家提交移动→「结束本轮行动」后，面板显示「已结束本轮行动，不可更改」。
6. **抽卡一次**：到达抽卡房间点「抽卡」后按钮置灰，再点报错。
7. **抽卡即时私密展示**：抽完立即出现「本次抽到（私密）：…」绿框。
8. **酒→果汁**：道具、酒窖产出、空投第 2 轮、饮品师文案全为「果汁」。
9. **饮品师多目标**：上方「使用果汁」选 2，技能区出现 2 个目标条可分配不同玩家。
10. **催眠不提前知道**：催眠师对 X 发动后，公共看板无任何提示；切到 X 在其行动前看不到被催眠，轮到 X 才显示「你被催眠」。
11. **被催眠轮到自己才显示强制移动**：X 行动面板顶部出现强制前往房间提示且不能另选目标。
12. **催眠 5 步不可达失败**：给催眠师选超过 5 步的目标房间，提交报错提示将失败。
13. **侦探只跟前序已行动**：侦探技能下拉只列顺位在它之前且已「已行动」的玩家。
14. **侦探看不到目标房间**：下拉只显示 `P# 昵称｜角色名（已行动）`，无房间。
15. **激光不立即公开扣血**：经过 102 的玩家提交后生命不变；结算后才 -1，公共日志结算阶段才出现。

# 禁闭逃杀 · 电子版 v0.1（可玩原型）

把根目录《禁闭逃杀》纸面规则做成可在网页游玩的电子桌游原型。本版本目标是**让流程跑通**：开房 → 进人 → 设置 → 6 轮（自由 / 行动 / 结算）→ 最终排名，而非追求完整自动结算。

> 规则唯一来源：`禁闭逃杀_规则手册.md`。所有规则数据已拆分到 `src/game/config/`，未写死在 UI 组件里。

## 技术栈

- Next.js 14（App Router）+ TypeScript + Tailwind CSS
- Zustand 状态管理
- localStorage 持久化 + 跨标签页 `storage` 事件同步（`src/store/storage.ts` 已预留 `StorageAdapter` 接口，便于 v0.2 接入 Supabase / Firebase 实时后端）

## 如何启动

```bash
npm install
npm run dev
```

浏览器打开 http://localhost:3000

生产构建：`npm run build && npm start`

## 如何创建 / 加入房间

- **创建房间**：首页填写房主昵称（可勾选「开发调试模式」允许少于 9 人开始）→ 创建 → 进入大厅，生成 6 位房间码。
- **加入房间**：首页输入房间码 → 加入。

### v0.1 的本地多人模型（重要）

v0.1 没有联机后端，房间数据存在**创建它的那个浏览器**的 localStorage 里：

- **同一浏览器多标签页**：共享房间数据，通过 `storage` 事件实时同步（适合一个标签页开 `/board` 投屏、另一个开 `/play`）。
- **单机热座测试**：大厅可在同一屏直接填满 9 个座位；用「我控制的玩家 / 切换身份」按钮切换视角，即可一人跑通整局，完成验收。
- 跨浏览器 / 跨设备的真实联机留待 v0.2 后端。

## v0.3 已实现（地图移动合法性与行动体验升级）

- **地图图结构**（`src/game/config/mapGraph.ts`）：对照 `禁闭逃杀_地图.png` 录入全部房间节点（楼层/列坐标/普通房间标记）与相邻边（同层横向、楼梯竖向、中央廊桥），含 `validateMapGraph()` 校验；地图规则集中配置，未写死在 UI。
- **路径工具**（`src/game/utils/movement.ts`，纯函数）：`getReachableRooms / findShortestPath / validateMove / getMovePreview / buildMoveContext`，BFS 按速度计算可达房间、最短路径与触发效果。
- **移动约束**：玩家不能再随便填终点——只能从系统计算的可达房间中选择；禁止原地停留；超出速度范围禁止提交并说明原因（含所需步数）。
- **特殊移动**：绳索/暗影不经楼梯竖向上下楼（每跨 1 层 1 步）、B105→B503 垃圾管道单向、B403 传送室到任意普通房间、202 停机坪直升机（资格=上一轮停留 202，仅可达 B101/101/B103/201/B105）、肾上腺素生效轮速度按 10 计算。
- **102 激光室**：路径经过/停留 102，提交移动时即时 -1（暗影免疫；改提交会回退旧伤害避免重复；结算不再重复扣血）。
- **玩家行动页升级**：显示当前位置/速度/状态/关键道具/直升机资格；地图绿色高亮可达房间并可点选；移动预览显示路径、步数、特殊移动、激光警告。
- **移动日志**：每次确认移动写入路径、步数、特殊移动、激光伤害日志。
- **行动提交结构升级**：`PlayerRoundAction` 增加 `path/stepsUsed/usedSpecialMove/triggeredEffects/laserDamageApplied`（旧数据字段可选，向后兼容）。
- **回归**：v0.2 的道具/战斗/毒气/水粮/暗影/火箭筒/最终排名均保持，`npm test` 共 28 项（含 v0.2 回归）全通过。

## v0.2 已实现（核心结算自动化 + 道具库存 + 地图）

- **地图接入**：`public/禁闭逃杀_地图.png`，`GameMap` 组件在玩家页 / 公共战况页 / 房主控制台显示；房间按楼层列表，高亮目标房间、当前位置、毒气楼层
- **道具系统**：全部 15 种道具配置（含 weight / weaponBonus / isGun / type）
- **库存系统**：房间初始库存（来自规则 14.1）、玩家手牌、全场消耗堆、停尸间遗物库存
- **抽卡**：常规随机抽卡（受抽卡上限与库存限制）、B503 垃圾场特殊抽卡（非垃圾最多保留 2 张）、金条额外选取、停机坪空投领取
- **负重**：`getInventoryWeight / getCarryLimit / isOverweight`，次元口袋无限负重、雇佣兵武器免负重；超重时阻止提交 + 丢弃面板
- **结算引擎**（`src/game/resolution/`，严格按规则 8.1 固定顺序）：
  - 房间效果（激光室 -1、手术室 2 人 +4、基因库三项 +1）
  - 战斗 / 乱斗 / 枪械压制（规则 9 完整算法）
  - 暗影吸血、累计吸血、死亡变暗影（遗物入停尸间）、复活（位置/生命/清零）
  - 火箭筒（目标房间存活玩家 -4，暗影免疫，持有可重复使用）
  - 毒气（投票产生楼层 + 按轮伤害，防毒面具/停机坪/复活当轮免疫，B501 特殊规则）
  - 水粮（第 2 轮起，交齐 0 / 缺一 -1 / 全缺 -2，餐厅与复活当轮免除）
  - 道具回血/状态（药片 +2、酒掷骰、肾上腺素下一轮生效）
  - 死亡/复活检查
- **房主结算预览**：「生成结算预览」逐步展示日志与生命变化，「确认应用结算」后写入状态并自动进入下一轮 / 最终结算
- **最终结算**：第 6 轮后金条兑换生命、排名（规则 17.3/17.4）、金魔方积分
- **测试**：`npm test` 运行 18 项核心结算纯函数测试（战斗/毒气/水粮/暗影/库存），全部通过

## v0.1 已实现

## 目录结构

```
src/
  app/                     页面与路由
    page.tsx               首页
    rules/page.tsx         规则页
    room/[roomCode]/       大厅 / play / board
  components/              UI 组件
  game/
    types.ts               核心数据类型
    inventory.ts           库存与负重工具
    gas.ts                 毒气房间判定（含 B501 特殊规则）
    config/                规则配置（唯一规则数据来源，含 initialStocks）
    engine/                流程引擎（创建/开局/提交/抽卡/阶段推进/房主修正）
    resolution/            结算引擎（战斗/毒气/水粮/暗影/排名等纯函数 + 预览编排）
  store/
    storage.ts             存储适配层（预留实时同步接口）
    useGameStore.ts        Zustand 全局状态（变更唯一入口 apply()）
scripts/selftest.ts        核心结算纯函数测试（npm test）
```

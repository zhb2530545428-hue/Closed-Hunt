## 3.3 道具与库存系统

### 3.3.1 玩家库存

玩家状态中必须有库存字段：

```ts
type PlayerState = {
  inventory: ItemId[]
}
```

如果当前用对象计数形式，也可以使用：

```ts
type Inventory = Record<ItemId, number>
```

但全项目要统一，不要混用。

### 3.3.2 房间库存

房间状态中需要保存当前库存。

```ts
type RoomState = {
  id: string
  inventory: Inventory
  isGasCleared?: boolean
}
```

### 3.3.3 消耗堆

游戏状态中增加：

```ts
discardPile: Inventory
consumedPile: Inventory
```

说明：

- 主动丢弃 / 被迫超重丢弃：留在当前房间；
- 使用后消耗：进入全场已消耗道具卡堆；
- 死亡玩家遗物：进入 B701 停尸间库存。

### 3.3.4 负重检查

实现函数：

```ts
getInventoryWeight(player): number
getCarryLimit(player): number
isOverweight(player): boolean
```

规则：

- 每张道具卡占用 1 点负重；
- 次元口袋本身占 1 张道具卡；
- 持有次元口袋后，负重上限视为无限；
- 失去次元口袋后立即重新检查负重；
- 雇佣兵的刀、手枪、霰弹枪不占负重；
- 饮品师的果汁规则在当前规则手册里没有实际道具“果汁”库存，v0.2 不强制实现果汁，只保留职业描述。

### 3.3.5 超重处理

当玩家因抽卡、交易、获得道具等导致超重时：

- UI 提示玩家必须丢弃；
- 玩家选择丢弃到不超过负重；
- 丢弃的道具留在当前房间；
- 未处理超重时，玩家不能提交行动或进入下一阶段。

v0.2 可以实现为：

```text
出现超重提示 → 玩家勾选要丢弃的道具 → 确认丢弃 → 继续游戏
```

- 不能超过该房间抽卡上限；
- 库存不足时，只能抽取剩余数量；

## 3.6 战斗 / 乱斗自动结算

### 3.6.1 触发条件

结算阶段中，若同一房间内有 2 名或以上存活玩家，触发战斗或乱斗。

- 2 名存活玩家：战斗；
- 3 名或以上存活玩家：乱斗；
- 暗影不计入人数；
- 暗影不参与战斗或乱斗。

### 3.6.2 战斗力计算

```text
战斗力 = 当前武力 + 武器加成 + 其他加成
```

v0.2 先实现：

```text
刀：+2，不是枪
手枪：+2，是枪
霰弹枪：+4，是枪
```

如果玩家持有多把武器，建议先按“所有持有武器加成都生效”实现，除非当前代码/规则另有明确限制。

### 3.6.3 普通战斗

没有枪械压制时：

1. 比较房间内所有参战玩家战斗力；
2. 战斗力最高者不扣血；
3. 多人并列最高，所有并列最高者不扣血；
4. 其他玩家扣除：最高战斗力 - 自己战斗力。

### 3.6.4 枪械压制

当房间内至少有 1 名持枪玩家时：

1. 先计算所有参战玩家战斗力最高值；
2. 所有并列最高战斗力玩家均不受战斗伤害；
3. 如果最高战斗力玩家全部为无枪玩家，则按普通战斗结算；
4. 如果最高战斗力玩家中存在持枪玩家，则：
   - 持枪玩家之间按普通战斗规则结算；
   - 非最高战斗力的无枪玩家，受到房间内最高持枪玩家的完整战斗力伤害。

### 3.6.5 手术室例外

B202 手术室：

- 若有 1 人：无事发生；
- 若有 2 人：结算阶段每人生命值 +4，且不发生战斗；
- 若超过 2 人：不手术，触发乱斗。

因此，resolveRoomEffects 或 resolveCombat 中需要对 B202 做特殊处理，避免 2 人时错误触发战斗。

## 3.7 毒气系统

### 3.7.1 毒气投票

每名存活玩家行动结束前提交 1 个楼层作为毒气投票。

规则：

- 结算阶段统计所有毒气票；
- 得票最高的楼层成为毒气楼层；
- 若多个楼层并列最高票，则所有并列楼层都成为毒气楼层；
- 已经成为毒气楼层的楼层不能再次被投票；
- 毒气不会自行消散；
- 暗影不能参与毒气投票。

B101 控制室若实现“本轮自己毒气投票 1 票视为 10 票”，需要计入投票权重。

### 3.7.2 毒气伤害表

```text
第 1 轮：-1
第 2 轮：-2
第 3 轮：-2
第 4 轮：-3
第 5 轮：-3
第 6 轮：-4
```

### 3.7.3 免疫毒气

以下情况不受毒气伤害：

```text
持有防毒面具
位于 202 停机坪
暗影
暗影复活后的当轮
位于已被 B101 控制室解除毒气的房间
```

### 3.7.4 B501 大仓库特殊毒气规则

B501 大仓库只有在 B4 和 B5 都成为毒气楼层后，才会成为毒气区域。

实现时不要简单地因为 B501 属于 B5 就立刻判定为毒气。

结算规则：

```text
上交 1 水 + 1 粮食：不扣血
只上交水或只上交粮食：-1 生命值
水和粮食都未上交：-2 生命值
```

免除水粮：

```text
本轮位于 B204 餐厅
暗影
暗影复活后的当轮
```

交互建议：

- 玩家行动阶段或结算前选择是否上交水/粮；
- 如果玩家选择上交但库存不足，系统自动按实际库存处理，并在日志中说明；
- 上交的水粮进入 consumedPile。

## 3.9 暗影、死亡与复活

### 3.9.1 生命值检查

死亡检查统一在结算阶段最后执行。

规则：

- 生命值不会低于 0；
- 结算过程中降到 0 的玩家，仍等到最后死亡检查；
- 死亡检查时生命值为 1 或以上：仍存活；
- 死亡检查时生命值为 0：成为暗影。

### 3.9.2 成为暗影

玩家成为暗影时：

- 公开该玩家进入暗影状态；
- 上交全部道具卡；
- 道具进入 B701 停尸间库存；
- 该玩家下一轮从 B701 停尸间出发；
- 生命值为 0；
- 没有武力；
- 没有负重；
- 继承死亡时速度。

### 3.9.3 暗影属性

暗影玩家：

```text
不能使用房间功能
不能抽取道具卡
不能参与毒气投票
不计入房间人数
不参与战斗或乱斗
不受毒气伤害
不需要上交水粮
不受激光室影响
不受火箭筒影响
```

### 3.9.4 暗影吸血

结算阶段的暗影步骤中：

- 若暗影与存活玩家位于同一房间，则每个暗影从每名存活玩家身上吸取 1 点生命值；
- 每名存活玩家受到的吸血伤害等于同房间暗影数量；
- 每个暗影分别记录自己的累计吸血量；
- 记录最后一次成功吸血所在房间。

玩家状态中建议增加：

```ts
shadowDrainCount: number
lastDrainRoomId?: string
reviveNextRound?: boolean
reviveProtectedRound?: number
```

### 3.9.5 暗影复活

规则：

- 暗影累计吸取生命值达到 2 点或以上时，满足复活条件；
- 暗影不会在吸血的同一结算步骤中立刻行动；
- 复活在下一轮开始时生效；
- 复活位置为最后一次成功吸血所在房间；
- 复活生命值等于累计吸血量；
- 复活后累计吸血量清零；
- 复活当轮必须移动，不能停留在复活房间；
- 复活当轮不需要上交水粮，不受毒气伤害。

## 3.10 火箭筒系统

火箭筒规则：

- 持有火箭筒的玩家每轮行动阶段可选择一个房间袭击；
- 结算阶段火箭筒步骤中结算；
- 该房间内每名存活玩家生命值 -4；
- 暗影不受影响；
- 公开火箭筒袭击的房间。

实现要求：

```ts
rocketTargets: Array<{
  playerId: string
  targetRoomId: string
  round: number
}>
```

若多个玩家使用火箭筒，按同一火箭筒步骤逐条处理即可。

火箭筒是否使用后消耗：规则表写作“武器 / 主动道具”，没有明确写使用后消耗。v0.2 不要擅自设为一次性消耗；先按“持有后每轮可使用”实现，并在代码注释中标明依据。

## 3.11 基础道具效果

### 3.11.1 药片

- 结算阶段道具回血 / 状态步骤使用；
- 生命值 +2；
- 不超过生命值上限；
- 使用后进入 consumedPile。

### 3.11.2 肾上腺素

规则：

- 结算阶段道具回血 / 状态步骤公开使用；
- 从下一轮开始生效；
- 下一轮速度变为 10；
- 下一轮无论受到多少伤害，生命值最多降至 1，不会死亡；
- 效果持续到下一轮结算阶段结束，随后速度恢复原状。

实现建议：

```ts
pendingAdrenalineRound?: number
adrenalineActiveRound?: number
baseSpeedBeforeAdrenaline?: number
```

伤害应用函数需要支持：

```ts
applyDamage(player, amount, reason)
```

当玩家处于肾上腺素保护轮时，生命值最低保留为 1。

### 3.11.3 酒

使用酒后掷 1 个六面骰：

```text
1：丢弃所有道具卡
2：无事发生
3：武力永久 +1
4：速度永久 +1
5：负重永久 +1
6：生命值 +2，不超过生命值上限
```

实现要求：

- 可以点击“使用酒”；
- 系统随机 1-6；
- 在日志里公开点数和效果；
- 使用后的酒进入 consumedPile。

### 3.11.4 金条

两种用途：

1. 抽卡时使用：可检视所在房间当前所有道具，并额外选择 1 张获得，不能选择金条；金条不可在 B206 金库和 B503 垃圾场使用；
2. 最终结算时：1 金条兑换 1 生命值，不超过生命值上限。

v0.2 必须实现最终结算兑换。

抽卡时使用金条可以实现基础版：

```text
玩家在可抽卡房间点击使用金条 → 显示该房间非金条库存 → 选择 1 张获得 → 消耗 1 金条
```

### 3.11.5 防毒面具

- 持有生效；
- 毒气结算时免疫毒气伤害；
- 不需要主动使用。

### 3.11.6 次元口袋

- 持有生效；
- 负重上限视为无限；
- 本身仍是一张道具卡；
- 失去后重新检查负重。

### 3.11.7 循环回收装置

规则：

- 每轮行动阶段可从全场已消耗道具卡中随机抽取 1 张获得；
- 获得后不能超过负重。

v0.2 可作为次优先级实现。若实现，必须从 consumedPile 抽取，不要从房间库存凭空生成。

## 3.12 最终结算与排名

第 6 轮结算完成后进行最终结算。

最终结算时：

- 存活玩家可以将手中金条兑换为生命值；
- 1 金条 = +1 生命值；
- 不能超过生命值上限。

排名优先级：

```text
1. 存活玩家排在暗影玩家之前
2. 存活玩家之间，生命值高者排名靠前
3. 存活玩家生命值相同，武力高者排名靠前
4. 暗影玩家之间，生前武力高者排名靠前
```

金魔方积分：

```text
第 1 名：9
第 2 名：8
第 3 名：7
第 4 名：6
第 5 名：5
第 6 名：4
第 7 名：3
第 8 名：2
第 9 名：1
```

如果只剩 1 名存活玩家，应提前结束并宣布胜者。

如果所有玩家都成为暗影，按规则手册的“全员暗影”规则处理。

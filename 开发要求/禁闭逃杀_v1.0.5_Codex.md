## 0. 本次任务性质

本次不是重做项目，也不是新增大版本功能，而是在当前 v1.0.x 基础上继续修复试玩过程中发现的规则漏洞、流程漏洞和 UI 误导问题。

本次版本名建议：

```text
v1.0.5：结算确认、操作室、催眠师、慈善家与日志整理修复包
```

核心修复点：

1. 修复 `B304 操作室 / 操作间` 无法重新分配自己基因点的问题。

## 1. 修复 B304 操作室：重新分配自己的基因点数

### 1.1 当前问题

玩家进入 `B304 操作室` 后，无法正确重新分配自己的基因点数。

规则手册中写明：

```text
B304 操作室：可立即重新分配自己的基因点数。
```

### 1.2 修复要求

当玩家本轮最终停留在 `B304 操作室`，且房间未被黑客关闭时，房间功能处应允许该玩家重新分配自己的基因点数。

必须满足：

- 只能重新分配自己的基因点数；
- 不影响生命值；
- 不影响职业带来的永久加成，除非当前代码已将职业加成合并到属性中，请谨慎处理；
- 速度不能被分配为 0；
- 初始 / 基础基因分配总和不能超过规则允许值；
- 分配完成后立即更新玩家面板；
- 写入玩家私密日志；
- 不写入公开日志具体数值，避免泄露玩家属性。

### 1.3 属性约束

根据此前用户确认：

- 速度永远不能为 0；
- 初始属性加和不能超过 10；
- 游戏中通过基因库、酒 / 果汁骰子、职业等永久加成可能导致总属性超过 10，这类额外加成不要被误删；
- 操作室应重新分配“可分配基因点”，不要吞掉永久加成。

请检查当前数据结构中是否区分：

```ts
baseForce / baseSpeed / baseLoad
bonusForce / bonusSpeed / bonusLoad
currentForce / currentSpeed / currentLoad
```

如果当前没有区分，请用最小风险方式修复，不要大规模重构。至少要避免操作室把职业 / 道具 / 事件获得的永久加成错误清零。

### 1.4 验收

- 玩家进入 B304 操作室；
- 房间未被黑客关闭；
- 出现重新分配基因面板；
- 速度不能调到 0；
- 总可分配点数不能超过允许值；
- 确认后玩家属性更新；
- 若 B304 被黑客关闭，则不能使用该功能。

## 2. 结算应用前：玩家必须确认资源使用选择

### 2.1 当前问题

房主点击“确认结算应用”时，有些玩家还没确认自己是否交水、粮、药、果汁、肾上腺素等资源，导致结算可能提前应用。

### 2.2 新规则要求

在房主确认应用结算前：

- 所有“手中有可在结算阶段处理的物资”的玩家，必须在自己的结算资源面板中点击确认；
- 不管玩家选择使用还是不使用，都必须点确认；
- 如果玩家手中完全没有这些可处理物资，可以不要求确认；
- 房主点击“确认结算应用”时，如果还有玩家未确认，不能应用结算；
- 系统应弹窗或明显提示：还有哪些玩家没有确认；
- 提示中使用角色名称，不要使用玩家 ID。

### 2.3 需要纳入确认的物资

至少包括：

```text
水
粮食
药片
果汁
肾上腺素
```

请根据当前代码里“结算阶段可使用 / 可上交”的资源列表统一处理，不要漏掉。

### 2.4 确认状态设计建议

建议增加类似状态：

```ts
type SettlementResourceConfirmation = {
  playerId: string;
  roundKey: string;
  hasConfirmableResources: boolean;
  confirmed: boolean;
  choices: {
    submitWater?: boolean;
    submitFood?: boolean;
    useMedicineCount?: number;
    useJuiceCount?: number;
    useAdrenaline?: boolean;
    // 按当前实现扩展
  };
};
```

要求：

- 每轮结算预览生成时初始化确认状态；
- 玩家修改选择后仍需点击确认；
- 玩家确认后可以在房主应用结算前修改吗？如果当前项目没有锁定机制，建议点击确认后锁定；若允许修改，则修改后必须重新确认；
- 房主应用结算前统一检查 `confirmed`。

### 2.5 房主提示

如果房主点击确认应用失败，提示示例：

```text
仍有玩家未确认结算资源选择：催眠师、富豪、雇佣兵
请等待他们确认后再应用结算。
```

注意：使用角色名称，不要用玩家 ID。

### 2.6 验收

- 有水 / 粮 / 药 / 果汁 / 肾上腺素的玩家未确认时，房主不能应用结算；
- 弹出缺失确认玩家列表；
- 玩家确认后，房主可以应用结算；
- 没有相关资源的玩家不阻塞结算；
- 不会重复扣除资源；
- 不会重复应用结算。

## 4. 本地热座随机生成玩家：避免极端基因

### 4.1 当前问题

随机生成角色时可能出现类似：

```text
0 / 10 / 0
10 / 0 / 0
0 / 0 / 10
```

这类极端基因不适合测试，也可能违反“速度永远不能为 0”的要求。

### 4.2 修复要求

本地热座随机生成玩家时：

- 速度不能为 0；
- 建议武力、速度、负重都至少为 1；
- 三项基础属性总和必须为 10；
- 避免某一项过高导致极端角色；
- 推荐每项范围为 `1-8`，更稳妥可以用 `2-6` 区间；
- 随机生成只用于测试，不改变正式玩家手动分配规则。

### 4.3 推荐算法

可使用以下策略之一：

策略 A：预设模板随机

```ts
const genePresets = [
  { force: 4, speed: 3, load: 3 },
  { force: 3, speed: 4, load: 3 },
  { force: 3, speed: 3, load: 4 },
  { force: 5, speed: 3, load: 2 },
  { force: 2, speed: 5, load: 3 },
  { force: 3, speed: 2, load: 5 },
  { force: 4, speed: 4, load: 2 },
  { force: 2, speed: 4, load: 4 }
];
```

策略 B：随机生成但强约束

```text
force >= 1
speed >= 1
load >= 1
force + speed + load = 10
max(force, speed, load) <= 7
```

优先推荐策略 A，便于测试稳定。

### 4.4 验收

连续随机生成多名玩家：

- 不出现速度为 0；
- 不出现 0/10/0 或类似极端分配；
- 总和始终为 10；
- 角色名称与身份显示正常。

## 5. 公开日志优化：按阶段分开，不要扎堆

### 5.1 当前问题

公开日志现在不够清晰，多个阶段信息扎在一起，玩家很难看懂。

### 5.2 修复要求

公开日志按阶段分块展示。

建议至少分为：

```text
首轮出生结算
自由阶段
行动阶段
结算阶段：房间事件
结算阶段：战斗 / 乱斗
结算阶段：暗影
结算阶段：火箭筒
结算阶段：毒气
结算阶段：水粮
结算阶段：道具 / 状态
结算阶段：死亡 / 复活
最终结算
```

注意：不是每个阶段都一定要显示。如果某阶段没有公开信息，可以显示“无公开事件”或直接折叠。

### 5.3 信息隐藏要求

继续遵守此前规则：

- 行动阶段不显示谁完成了行动；
- 不显示玩家移动路径；
- 不显示玩家位置；
- 战斗 / 乱斗只显示哪个房间发生了战斗或乱斗；
- 不显示谁打了谁；
- 不显示谁在房间里；
- 不显示每个人因战斗扣了几点血；
- 毒气只显示哪个楼层成为毒气楼层，不显示票数，不显示谁在哪里因毒气扣血；
- 火箭筒公开袭击房间，但不要暴露房间内具体玩家；
- 水粮不要公开谁缺水缺粮导致扣血；
- 房间效果只对进入该房间且应知情的玩家显示；
- 最终统一显示所有玩家当前生命值。

### 5.4 日志结构建议

建议对日志事件增加：

```ts
type LogVisibility = 'public' | 'private' | 'host';

type LogPhase =
  | 'initial_spawn_resolution'
  | 'free_phase'
  | 'action_phase'
  | 'resolution_room'
  | 'resolution_combat'
  | 'resolution_shadow'
  | 'resolution_rocket'
  | 'resolution_gas'
  | 'resolution_supply'
  | 'resolution_item_status'
  | 'resolution_death_revival'
  | 'final';
```

UI 根据 `phase` 分组展示，不要只按时间一条一条堆。

### 5.5 验收

- 公开日志按阶段分块；
- 不泄露隐藏信息；
- 玩家能清楚看出本轮发生了哪些公开事件；
- 私密日志仍能让本人知道自己遭遇了什么；
- 房主裁判视图可以保留更详细信息，但要明确只给房主看。

## 6. 催眠师技能：回合前询问 + 自我催眠锁定移动

### 6.1 当前问题

催眠师技能存在两个问题：

1. 催眠师选择催眠自己时，自己后续仍可能选择要去哪个房间；
2. 技能发动时机应改为：如果场上有催眠师，除首轮外，每回合行动阶段所有玩家开始行动之前，先询问催眠师是否使用技能，用在谁身上。

### 6.2 正确流程

除首轮外，每回合进入行动阶段前：

1. 系统检测场上是否存在存活的催眠师；
2. 如果有催眠师且本回合未使用过技能，先进入“催眠师技能询问”步骤；
3. 催眠师选择：
   - 不使用；或
   - 使用技能；
4. 如果使用技能，需要选择：
   - 催眠目标玩家，可以是自己；
   - 指定目标房间；
5. 系统判断：该被催眠玩家是否能在 5 步内到达指定房间；
6. 如果 5 步内无法到达，催眠失败；
7. 如果催眠成功，记录一个隐藏的强制移动计划；
8. 之后正常进入行动阶段，按顺位行动。

### 6.3 被催眠者轮到自己行动时

如果当前行动玩家有本轮催眠强制移动计划：

- 系统提示该玩家：你本轮受到催眠影响，将前往指定房间；
- 该玩家不能选择其他房间；
- 其他房间不可点击 / 不可选；
- 目标房间显示特殊标记；
- 玩家只需要确认执行强制移动；
- 成功移动后可以正常触发该房间的可用效果，除非房间被黑客关闭；
- 该逻辑对催眠师催眠自己也同样生效。

### 6.4 信息隐藏

- 催眠师使用技能不应立即公开；
- 被催眠玩家在轮到自己行动前，不应提前知道；
- 其他玩家不应知道谁被催眠、被催眠去哪里；
- 公开日志不显示催眠目标和目的地；
- 私密日志只给相关玩家看；
- 房主裁判视图可以看到完整信息。

### 6.5 5 步判断

按照用户确认：

- 判断固定使用 5 步；
- 与被催眠玩家自身速度无关；
- 5 步内到不了则失败；
- 成功后强制移动到指定房间。

请复用当前地图路径算法，但传入 `maxSteps = 5`，不要使用目标玩家当前速度。

### 6.6 首轮例外

首轮只做出生房间选择后的第一次战斗结算，不进入正常行动阶段，不询问催眠师技能。

### 6.7 验收

- 有催眠师时，第 1-6 轮行动阶段前先询问是否使用技能；
- 首轮不询问；
- 催眠师可以选择自己作为目标；
- 催眠成功后，目标玩家轮到自己行动时房间选择被锁定；
- 目标房间以特殊标记显示；
- 其他房间不能点击；
- 5 步到不了时失败；
- 公开日志不泄露催眠信息。

## 7. 慈善家技能：结算预览阶段使用，只能赠出一个道具

### 7.1 当前问题

慈善家“赠出道具”栏可以把相同道具一起送出，导致不符合规则。

用户本次补充确认：

- 慈善家只能赠送一个道具；
- 不可以把相同道具作为一组一起送出；
- 该技能比较特殊，应在生成结算预览，也就是玩家选择交不交资源的页面时使用；
- 慈善家比其他玩家多一条“是否使用技能”。

### 7.2 正确流程

当进入结算预览 / 玩家资源确认阶段时：

如果某玩家是存活慈善家，且本轮 / 本局仍可使用该技能，则该玩家的私密结算资源面板里额外显示：

```text
慈善家技能：是否赠出 1 张道具？
```

选择使用后：

1. 选择 1 名其他存活玩家作为赠予目标；
2. 从自己当前道具中选择 1 张具体道具实例；
3. 确认赠出；
4. 被赠予玩家可以短暂超过负重；
5. 被赠予玩家必须公开选择将 1 点可转移基因永久转移给慈善家；
6. 不能选择数值为 0 的基因；
7. 整场对局中，1 名玩家只能被赠予 1 次。

### 7.3 道具选择必须按实例选择

不能这样：

```text
赠出：水 × 3
赠出：刀 × 2
```

必须这样：

```text
赠出：水 #item_abc123
```

UI 上不一定展示内部 ID，但代码内部必须按单张道具实例处理。

如果 UI 为了友好显示，可以写：

```text
水（第 1 张）
水（第 2 张）
刀（第 1 张）
```

但实际提交时只能提交一个 itemId。

### 7.4 技能确认状态

慈善家技能属于结算预览阶段的一部分，因此：

- 慈善家如果有可结算资源或可用技能，也必须点击确认；
- 如果慈善家选择使用技能但没有完成目标 / 道具选择，不能确认；
- 房主应用结算前要检查慈善家的选择状态；
- 慈善家不使用技能也要明确确认“不使用”。

### 7.5 公开与私密信息

- 赠出道具和基因转移属于公开事件还是私密事件，请以当前规则手册和已有实现为准；
- 如果规则不明确，优先不要在公开日志中泄露过多细节；
- 至少相关双方必须知道具体道具和基因变化；
- 房主裁判视图必须完整可见；
- 如果无法确认公开程度，请写入 `RULE_QUESTIONS.md`。

### 7.6 验收

- 慈善家在结算预览 / 资源确认页面看到额外技能栏；
- 可以选择不使用；
- 使用时只能选择一个目标；
- 只能选择一张具体道具；
- 不能一次赠送多个相同道具；
- 被赠予玩家按规则选择转移哪 1 点基因；
- 已被赠予过的玩家不能再次成为目标；
- 房主应用结算前会等待慈善家确认。

## 11. 鼠标验证清单

请确保用户可以按以下方式验证：

### 11.1 操作室

1. 开局进入游戏；
2. 让某玩家进入 B304 操作室；
3. 确认出现基因重新分配面板；
4. 尝试把速度调成 0，应被禁止；
5. 正常分配后确认，属性更新。

### 11.2 结算资源确认

1. 给几个玩家水 / 粮 / 药 / 果汁 / 肾上腺素；
2. 进入结算预览；
3. 让其中一名玩家不确认；
4. 房主点击应用结算；
5. 应失败并提示未确认玩家角色名称；
6. 所有人确认后，应用成功。

### 11.3 重置按钮删除

1. 检查热座 / 房主 / 结束页；
2. 不应再看到“重置本局”按钮。

### 11.4 随机玩家

1. 连续随机生成多名玩家；
2. 不出现 0/10/0；
3. 不出现速度 0；
4. 总和始终为 10。

### 11.5 公开日志

1. 完整跑一轮；
2. 查看公开日志；
3. 日志按阶段分开；
4. 不显示隐藏信息。

### 11.6 催眠师

1. 场上有催眠师；
2. 首轮不询问技能；
3. 第 1 轮行动前询问是否使用技能；
4. 催眠自己并指定房间；
5. 轮到自己时不能选其他房间，只能确认强制移动。

### 11.7 慈善家

1. 场上有慈善家；
2. 进入结算预览 / 资源确认页面；
3. 慈善家看到额外技能栏；
4. 选择使用技能；
5. 只能选一个目标；
6. 只能选一张具体道具；
7. 不能一次赠出多个相同道具；
8. 被赠予玩家完成基因转移选择；
9. 房主应用结算成功。

## 1. 项目目标

本版本要做的是：

```text
《禁闭逃杀》电子版 v0.1 可玩原型
```

核心目标不是做精美游戏，而是先做一个可以真实跑流程的网页桌游系统：

```text
创建房间 → 9 名玩家加入 → 选择职业 → 分配基因点 → 选择出生房间
→ 开始游戏 → 每轮自由阶段 / 行动阶段 / 结算阶段
→ 玩家提交行动 → 系统记录公开信息 → 推进 6 轮 → 显示最终排名入口
```

最终效果应该是：

1. 任意玩家打开网页即可创建或加入房间。
2. 玩家用房间码进入同一局游戏。
3. 每名玩家有自己的私密面板。
4. 公共页面显示当前轮次、阶段、公开日志和玩家公开生命值。
5. 游戏可以从第 1 轮推进到第 6 轮。
6. 当前版本允许部分复杂结算由主持人手动确认，但系统必须有清晰的数据结构和结算入口，为后续自动化做准备。

## 8. 游戏流程要求

### 8.1 开局

房主点击开始游戏后：

1. 校验玩家数和准备状态。
2. 初始化第 1 轮。
3. 每名玩家当前位置设为其出生房间。
4. 每名玩家生命值为 10。
5. 当前阶段进入 `FREE`。
6. 写入公开日志：

```text
游戏开始。第 1 轮自由阶段开始。
```

### 8.2 自由阶段

显示说明：

```text
玩家可以交流、交易顺位卡和道具卡。
自由阶段结束前需检查负重。
```

v0.1 暂时不强制实现交易系统。  
但界面上可以预留“交易功能后续版本开放”。

### 8.3 行动阶段

进入行动阶段时：

1. 为存活玩家随机生成顺位卡。
2. 按顺位展示行动顺序。
3. 玩家逐个或同时提交行动都可以，v0.1 建议允许同时提交。
4. 所有存活玩家提交后，房主可以进入结算阶段。

暗影玩家：

- 可以提交移动终点
- 不提交毒气投票
- 不使用房间功能
- 不抽道具

### 8.4 结算阶段

结算阶段必须显示规则中的固定结算顺序：

```text
1. 房间效果
2. 战斗
3. 暗影
4. 火箭筒
5. 毒气
6. 水粮
7. 道具回血 / 状态
8. 死亡 / 复活检查
```

v0.1 可以把每一项做成“待确认结算卡片”。

每张卡片显示：

- 该步骤名称
- 系统能自动判断出的信息
- 主持人备注输入框
- 确认按钮

确认完全部 8 个步骤后，允许进入下一轮。

### 8.5 毒气投票

v0.1 必须自动统计毒气投票。

规则：

- 暗影不参与毒气投票
- 已经成为毒气楼层的楼层不能再次被投票
- 得票最高楼层成为毒气楼层
- 若并列最高，所有并列楼层都成为毒气楼层

结算时写入公开日志：

```text
第 X 轮毒气投票结果：B3 成为毒气楼层。
```

如果多个并列：

```text
第 X 轮毒气投票结果：B3、B5 并列最高，均成为毒气楼层。
```

### 8.6 进入下一轮

第 1-5 轮结算结束：

```text
currentRound += 1
currentPhase = "FREE"
清空玩家 submittedAction
写入公开日志
```

第 6 轮结算结束：

```text
currentPhase = "GAME_OVER"
显示最终排名入口
```

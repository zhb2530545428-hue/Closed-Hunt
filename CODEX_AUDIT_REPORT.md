# 《禁闭逃杀》Codex 审计报告

## v1.0.6 审计记录

审计版本：v1.0.6  
执行方：Codex  
审计范围：RULE_QUESTIONS 清理、黑客关闭房间时点、餐厅/手术室结算失效、慈善家公开日志、饮品师多瓶果汁去重、暗影使者恢复、预言家能力、顺位卡交易校验与回归测试。

### v1.0.6 已修复

1. 黑客关闭房间时点
   - 新增 `closedRoomRecords`，记录关闭房间、轮次、黑客玩家与行动顺位。
   - 行动阶段即时效果通过 `isRoomFunctionDisabledForAction` 判断，只影响黑客行动之后进入或使用该房间功能的玩家。
   - 结算阶段效果通过 `isRoomFunctionDisabledForResolution` 判断，餐厅、手术室只要本轮结算前被关闭就统一失效。
   - 旧存档仍兼容 `closedRooms`，缺少顺位记录时按保守关闭处理。

2. 餐厅 / 手术室关闭修正
   - B204 餐厅被关闭后，结算阶段不再提供免水粮；餐厅内玩家按自身水粮选择正常扣血。
   - B202 手术室被关闭后，2 人不再获得手术 +4，也不再保留“不战斗”保护，照普通房间进入战斗/乱斗逻辑。

3. 慈善家公开日志
   - 慈善家赠予公开日志改为公开双方角色名：`慈善家向【角色名】赠予了 1 张道具。`
   - 仍不公开双方房间、移动路径、具体道具名或手牌信息。

4. 饮品师多瓶果汁
   - 结算资源确认时校验：使用 N 瓶果汁必须选择 N 个目标，且目标互不相同。
   - 校验失败时直接拒绝确认，提示同一目标不能被多瓶果汁重复作用。

5. 暗影使者恢复
   - 恢复量改为本轮暗影实际造成的生命损失点数。
   - 目标生命不足导致理论吸血超过实际扣血时，暗影使者只按实际吸到的点数恢复，且不超过生命上限。

6. 预言家能力
   - 预言家不再拥有意见领袖的额外毒气票权。
   - 职业配置文案已删除“决定顺位”和额外票权描述，保留死亡预告、+2 待分配基因和 +1 生命恢复。

7. 顺位卡交易
   - 继续沿用并验证“只允许交换”的交易模型。
   - 接受交易后执行 `validateTurnOrderCards()`，确保每名存活在座玩家仍且仅有 1 张顺位卡。

8. RULE_QUESTIONS 清理
   - 已删除本轮明确回答的历史问题。
   - 仅保留慈善家超重复检时机、水粮公开程度、远程模式服务端裁剪 3 个仍待确认问题。

### v1.0.6 验证

- `npm test`：113 通过，0 失败。
- `npm run build`：通过，Next.js 生产构建成功。
- 鼠标验证：通过浏览器创建房间、随机生成 9 名玩家、开始游戏、生成并确认首轮出生战斗预览；确认后进入“第 1 轮 / 自由阶段”，未出现“重置本局”按钮，公开日志按阶段分块展示。

# 《禁闭逃杀》v1.0.5 Codex 审计报告

审计版本：v1.0.5  
执行方：Codex  
审计范围：结算确认、B304 操作室、催眠师行动前询问、慈善家结算赠予、公开日志分组、本地热座随机基因、重置按钮移除、信息隐藏与重复结算风险。

## 已修复问题

1. B304 操作室
   - 玩家本轮最终停留在 B304 且房间未被黑客关闭时，可在私密面板重新分配自己的当前基因总和。
   - 速度最低为 1；三项总和必须等于当前总基因，避免吞掉职业、基因库、果汁等已获得的永久加成。
   - 结果只写入本人私密日志，不公开具体数值。

2. 结算确认门槛
   - 增加 `settlementConfirmations`，按轮记录需要确认的玩家。
   - 持有水、粮食、药片、果汁、肾上腺素，或慈善家有可用赠予技能的存活玩家，必须在结算面板点击确认。
   - 房主确认应用结算前会检查未确认名单；未确认时阻止应用，并提示角色名称列表。
   - 玩家重新确认会清掉旧结算预览，避免套用过期资源选择。

3. 删除重置本局
   - 删除房主控制台“重置本局”按钮。
   - 删除 `resetGameToLobby` 引擎函数和对应自测。
   - 保留创建房间、返回大厅、本地测试填充/随机生成玩家等正常开局能力。

4. 本地热座随机基因
   - 本地测试随机生成改为模板池：`4/3/3`、`3/4/3`、`3/3/4`、`5/3/2`、`2/5/3`、`3/2/5`、`4/4/2`、`2/4/4`。
   - 三项总和恒为 10，速度永不为 0，不再生成 `0/10/0` 等极端角色。

5. 公开日志分块
   - 新增 `logPhase`，结算落地日志按首轮出生结算、自由阶段、行动阶段、房间事件、战斗/乱斗、暗影、火箭筒、毒气、水粮、道具/状态、死亡/复活、最终结算分组展示。
   - 公开日志继续不显示玩家位置、路径、催眠目标、逐人伤害、毒气票数、水粮扣血来源等隐藏信息。
   - 公共战况的玩家列表也已隐藏当前位置、移动目标、负重和手牌，避免公开看板绕过日志脱敏。

6. 催眠师流程
   - 进入行动阶段时，若有可行动的存活催眠师，先生成行动前询问。
   - 催眠师可选择不使用，或选择目标玩家和目标房间；目标可为自己。
   - 使用成功后写入 `pendingHypnosis`，目标轮到行动时只可确认强制目标房间；自我催眠同样被锁定。
   - 5 步判断固定使用普通移动距离，与目标速度无关。

7. 慈善家流程
   - 慈善家技能移入结算资源确认面板。
   - 慈善家必须明确选择使用或不使用；使用时必须选择 1 名目标和 1 张具体手牌。
   - 代码记录 `giveItemIndex`，结算时只移走该序号对应的一张道具，不会把同名道具成组赠出。
   - 赠予细节进入房主/双方私密日志；公开日志只保留不暴露位置的概括事件。

## 审计发现并已修复

- 房主可能在玩家资源未确认时应用结算：已阻止。
- 结算预览生成后玩家重新确认可能导致预览过期：已在重新确认时清空旧预览。
- 操作室普通房间功能缺少实际入口：已补齐。
- 催眠师自我催眠后仍可改选房间：已统一走强制移动锁定。
- 行动阶段仍显示催眠师/慈善家旧技能位置：已迁移到新时机。
- 本地随机基因可能极端：已改为模板。
- 公开日志扎堆：已分组展示。
- 公共战况玩家列表泄露位置/手牌：已改为只展示顺位、角色、昵称、状态和生命值；完整信息仅保留在房主裁判区和玩家私密面板。

## 暂未高风险重构

1. 客户端级信息隐藏
   - 当前仍是本地热座/房主裁判信任模型，完整房间状态存在客户端。
   - 若未来用于不可信远程对局，需要服务端按身份裁剪快照；本次未做架构级改造。

2. 玩家属性未拆分 base/bonus
   - 当前数据结构仍只有 `force/speed/load`。
   - 操作室采用“重分配当前总和”的低风险方案，避免清零永久加成；如需精确区分基础点和永久加成，需要后续数据迁移。

3. 慈善家赠予公开程度
   - 本次采取保守公开，只公开概括事件，具体道具和双方细节放在私密/房主日志。
   - 是否需要公开双方角色名或道具名称已写入 `RULE_QUESTIONS.md`。
